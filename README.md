1.将文档切块后，使用llm对chunck提出用户可能会问道的问题，将这些问题存入
向量数据库中，当用户查询时，使用用户的问题先匹配预先存入的问题，如何相关性高，
在找到问题背后的完整文档。 （非对称检索的难点：在标准 RAG 中，用户输入的 Query 通常很短（如“如何申请年假？”），
而文档库里的知识块通常很长且是陈述句。由于两者的文本长度、句式和表达意图差异巨大，Embedding 模型在计算它们之间的向量相似度时，效果往往不够理想[1][2]。
对称检索的优势：使用该技术后，实际检索变成了**“用户问题”与“假设问题”**的匹配（Query-to-Query）。由于两者都是简短的疑问句，属于同一文本域，
这种对称性更符合绝大多数 Embedding 模型的训练分布，从而能显著提升检索的准确率）
2.创建子查询
3.简化复杂用户查询的另一种方法是创建回溯提示。这种技术包括使用 LLM 将复杂的用户查询抽象为"回溯问题"**。然后，向量数据库利用这些回溯问题来检索最相关的文档块。最后，LLM 根据这些检索到的文档块生成更准确的答案。
